ภาพรวม มุมมองผู้บริหาร มุมมองพนักงาน ทะเบียนกฎหมาย รายการติดตาม สรุปตามหมวด สรุปแนวโน้ม คุณภาพข้อมูล เกี่ยวกับรายงาน ผู้ดูแลระบบ

การแมปโครงสร้างเวิร์กบุ๊ก

ชีต โมดูลในระบบ หมายเหตุ
Raw_Data_Copy ทะเบียนกฎหมายหลักและตารางรายละเอียดสำหรับพนักงาน เป็นแหล่งข้อมูลหลักสำหรับ 74 แถวรายละเอียด
Data_Helper ชั้นการปรับมาตรฐานข้อมูลในโค้ด สะท้อนเป็นข้อมูลปี เดือน-ปี สถานะที่ปรับมาตรฐานแล้ว สถานะการบังคับใช้ และธงการปฏิบัติตาม
Summary_KPI การ์ด KPI สำหรับผู้บริหาร คำนวณจากฝั่งเซิร์ฟเวอร์เพื่อให้ตรวจสอบที่มาของ KPI ได้ชัดเจน
Summary_Category หน้าสรุปความเสี่ยงตามหมวดและส่วนผู้บริหาร แสดงจำนวนรวม รายการที่ยังมีผล ดำเนินการแล้ว อยู่ระหว่างทบทวน ประเมินแล้ว และรายการ Active ที่ยังค้าง
Summary_Trend หน้าสรุปแนวโน้มและส่วนกราฟ ใช้วันที่มีผลสำหรับการกระจายตัวรายปีและรายเดือน
Action_Backlog หน้ารายการติดตามและพาเนลสรุป ใช้ติดตามงานสำหรับรายการ Active ที่ยังค้าง การทบทวนรายการที่ไม่ใช้บังคับ และประเด็นคุณภาพข้อมูล
Workbook_Guide หน้าเกี่ยวกับรายงาน ใช้เป็นแนวทางอธิบายสมมติฐานและการตีความรายงาน
Dashboard บริบทของเวิร์กบุ๊กและข้อมูลอ้างอิงรายงาน แสดงเป็นบริบทโครงสร้างเวิร์กบุ๊กในหน้าเกี่ยวกับรายงาน

บริบทจากเวิร์กบุ๊ก

  • คู่มือเวิร์กบุ๊ก
  • วัตถุประสงค์ ไฟล์นี้ใช้สรุปผล Legal Dashboard (LD) จากข้อมูลจริงในทะเบียนกฎหมาย โดยคงโครงสร้างหลักจากไฟล์ตัวอย่างและปรับรูปแบบเป็น Modern Executive
  • คำอธิบายรายงาน ประจำเดือน Current month snapshot: เดือน มีนาคม | ไม่มี baseline เดือนก่อนหน้าในไฟล์นี้
  • หลักการตีความ ใช้ Compliance Implementation Status + Legal Validity Status เป็น KPI หลัก และมอง Active but not Implemented เป็น priority observation
  • แดชบอร์ด หน้าสรุปผู้บริหาร: KPI cards, charts, executive highlights, backlog preview และ recommendations
  • สรุป KPI คัดลอกข้อมูลต้นทางจากไฟล์ legal register ฉบับปัจจุบัน
  • สรุปตามหมวด แปลงข้อมูลเพื่อใช้คำนวณ status, validity, pending, blank date และ data issue flags
  • สรุปแนวโน้ม ตาราง KPI หลักสำหรับ dashboard cards และ executive review
